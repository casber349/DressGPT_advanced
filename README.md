# DressGPT Advanced 標註主控台

本專案為 DressGPT 核心數據管道的專用進階標註工作台。透過響應式前端網頁與高效的動態階梯式積分推薦系統，協助開發者與標註員針對服飾影像進行精確的風格分類、多維度屬性強度評估，並將清洗後的結構化標註數據持久化至 MySQL 資料庫中，為後續的機器學習模型訓練提供高質量的監督訊號。

## 🚀 核心功能特色

* 雙軌制並行評分面板：前端同時整合「左側正向加分屬性」與「右側負向扣分缺陷」兩大區塊，各提供 4 個動態槽位，點選下拉選單即可帶出資料庫對應的標籤。
* 階梯式權重積分演算：強度滑桿範圍為 0.0 至 10.0。系統依據滑桿數值自動判定段位並加權計算：
  - 弱 (C 級)：0.1 ~ 4.9 分，權重乘數為 1
  - 中 (B 級)：5.0 ~ 7.9 分，權重乘數為 2
  - 強 (A 級)：8.0 ~ 10.0 分，權重乘數為 3
* 全新線性映射建議分數：系統以 5.00 分為初始基準線。不論正向加分或負向扣分，總權重積分每變動 5 分，建議分數區間即連動調整 0.25 分。最終分數範圍安全限制在 0.00 至 10.00 之間，大幅提升標註數據的一致性。
* 即時同步與影像管理：支援前端直接動態新增自訂主風格與強度標籤至後端資料庫。影像切換支援快捷鍵、ID 精確跳轉與前後翻頁，並在儲存時提供視覺化邊框閃爍回饋。

## 💾 資料庫架構說明

專案採用 MySQL 關係型資料庫，核心結構包含以下四大資料表：

* 主風格清單表 (label_styles)
  - 欄位：style_name (主鍵，如：日系、復古、街頭)
  - 用途：管理前端主風格下拉選單的元資料

* 屬性標籤清單表 (label_attributes)
  - 欄位：attr_name (主鍵，如：性感、甜美、剪裁不佳)
  - 欄位：is_positive (布林值，1 代表正向優點，0 代表負向缺陷)
  - 用途：管理左右雙軌槽位選單的元資料庫

* 照片標註主資料表 (dress_dataset)
  - 欄位：image_id (主鍵，對應圖片檔名)
  - 欄位：gender (性別分類，預設 male)
  - 欄位：season (季節屬性，預設 spring_autumn)
  - 欄位：formality (正式度，預設 casual)
  - 欄位：image_path (靜態影像相對路徑)
  - 欄位：overall_score (最終定案分數)
  - 欄位：main_style (主風格外鍵)

* 照片多值屬性關聯表 (image_attributes)
  - 欄位：image_id (複合主鍵/外鍵)
  - 欄位：attr_name (複合主鍵/外鍵)
  - 欄位：attr_value (浮點數，存放 0.0 ~ 10.0 的滑桿強度)
  - 用途：儲存每張照片所勾選的所有細項標籤強度，採一對多關聯

## 🛠️ 環境建置與初始化

1. 複製本機專案並設置環境變數：
   在專案根目錄建立 .env 檔案，並配置你的資料庫連線資訊：
   - DB_HOST=你的資料庫IP
   - DB_USER=你的帳號
   - DB_PASSWORD=你的密碼
   - DB_NAME=dress_gpt_advanced

2. 資料庫初始化：
   開啟 MySQL 用戶端，依序匯入專案附帶的結構定義檔：
   - 執行 construct_db.sql 以動態建立資料庫與上述四大核心資料表。

3. 批次匯入初始影像：
   將所有待標註的服飾 .jpg 照片放入 static/dataset_images/ 目錄中。
   執行 extend_dataset.py，腳本會自動掃描該資料夾，並透過 INSERT IGNORE 機制安全地將未登記的 image_id 預載入資料庫，不破壞既有數據。

4. 啟動標註服務：
   執行 app_labeling.py 啟動 Flask 後端，於瀏覽器開啟 http://127.0.0.1:9527 即可開始作業。
